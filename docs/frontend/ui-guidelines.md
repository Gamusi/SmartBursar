# Frontend UI Guidelines

## Guiding principle

The interface should be simple, predictable, and easy for school staff to use.

## Design goals

- Clear primary actions
- Minimal input steps
- Visible status and confirmation after saving
- Consistent layout across screens

## Component rules

- Use simple cards, tables, and forms.
- Avoid nested modals or hidden functionality.
- Keep buttons and labels explicit.
- Keep information density moderate for low-resolution screens.

## Performance

- Load only what is needed for the current screen.
- Avoid large data fetches on initial load.
- Keep the bundle small and avoid unnecessary dependencies.

## Accessibility

- Ensure buttons and form fields are labeled.
- Use adequate color contrast.
- Prefer keyboard-friendly controls where possible.
