# SmartBursar Requirements

## Primary product requirements

- Offline-first operation with local data storage
- Simple, fast UI for school finance tasks
- Reliable student registration and enrollment tracking
- Payment collection with official receipt generation
- Expense and bank deposit recording
- Automatic cashbook entries from financial events
- Reports for daily collections, balances, and terms
- PDF output for receipts and reports
- Basic role-based access for bursar and administrative tasks
- Encrypted local backup and restore

## Non-functional requirements

- Works on low-spec Windows machines
- Minimal dependencies and straightforward deployment
- Easy to understand architecture and code
- Clear documentation of business decisions
- Maintainable by a single developer

## Constraints

- No internet dependency required for core workflows
- Target Windows 10 as primary platform, with optional Linux/macOS support
- Prefer simple storage and APIs over complex abstractions
